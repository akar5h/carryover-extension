CarryOver — Privacy Policy

Last updated: June 2026

CarryOver is a Chrome browser extension that compresses AI conversation context into portable checkpoints. This policy explains what data the extension accesses and what it does with it.

Data Accessed and Stored

  • OpenAI API Key. If you enter an API key in the extension popup, it is stored locally in your browser using chrome.storage.local. It is never transmitted to any server controlled by the extension developer. The key is used only when you trigger compression and the in-chat path is unavailable.
  • Conversation text. The extension reads visible conversation text from claude.ai and chatgpt.com tabs you have open. This is used locally to count tokens and build the compression prompt. The text is not sent to any server other than the AI platform you are already using (Anthropic or OpenAI) through your own account.
  • Pending checkpoint. After compression, the checkpoint is held temporarily in chrome.storage.session to auto-paste it into the new chat tab. This data is cleared after insertion or when the browser session ends.

Data Sharing

The extension does not collect, store on external servers, transmit, or share any personal data with the extension developer or any third party. All processing occurs locally in your browser or through your own AI platform account using your own API credentials.

Permissions Used

  • activeTab — detect the active AI platform tab.
  • scripting — inject the context ring and stats panel into supported pages.
  • clipboardWrite — copy the compressed checkpoint to your clipboard.
  • storage — persist your API key and pass the checkpoint between tabs.
  • Host permissions (chatgpt.com, claude.ai) — required for content script injection.
  • Host permissions (api.openai.com) — used by the optional background worker when calling OpenAI with your key.
